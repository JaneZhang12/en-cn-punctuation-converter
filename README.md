# 英→中 标点智能转换工具

将英文半角标点一键智能转换为中文全角标点。纯前端实现，无需安装，打开即用，数据不离开浏览器。

---

## 解决什么问题？

从 AI 对话、翻译工具或其他来源复制中文文本到 Word 文档时，标点符号经常是英文半角形式：

```
她说:"这是一个'测试'文本."
```

本工具可以将其智能转换为规范的中文全角标点：

```
她说："这是一个'测试'文本。"
```

无需用户在Word文档中进行查找替换的动作，即可一键完成符号转换。

## 功能特性

### 🎯 智能引号配对

核心便利点在于英文直引号 `"` 不分前后，工具通过上下文分析自动判断前引号（`"`）和后引号（`"`），支持双引号和单引号两种类型。针对英文缩写中的撇号（如 `don't`、`it's`）会自动跳过，避免误转换。

### ✅ 自由选择转换规则

除引号外，还支持以下标点的可选转换，按需勾选：

| 标点 | 转换 | 默认 |
|------|------|------|
| 双引号 | `"` → `""` | 开启 |
| 单引号 | `'` → `''` | 开启 |
| 逗号 | `,` → `，` | 关闭 |
| 句号 | `.` → `。` | 关闭 |
| 冒号 | `:` → `：` | 关闭 |
| 分号 | `;` → `；` | 关闭 |
| 感叹号 | `!` → `！` | 关闭 |
| 问号 | `?` → `？` | 关闭 |
| 括号 | `()` → `（）` | 关闭 |
| 破折号 | `--` → `——` | 关闭 |

### 🛡️ 智能环境检测

转换非引号标点时，工具会检测前后字符环境，自动跳过不应转换的场景：

- 数字中的小数点和千分位逗号不会被转换（如 `3.14`、`1,000`）
- 英文语境中的标点不会被转换（如 `e.g.`、`Hello, world`）
- 时间格式中的冒号不会被转换（如 `10:30`）

### 📂 两种使用方式

**手动粘贴**：将文本粘贴到输入框，点击转换，查看高亮差异后复制或导出。

**文件直传**：直接上传 `.txt` 或 `.docx` 文件，自动转换标点后下载同格式文件，保留原文档的所有排版格式。上传后会弹出扫描结果弹窗，预览各类标点的转换数量，确认后再执行转换。

### 📤 多种输出方式

- **复制到剪贴板**：一键复制转换后的纯文本
- **导出 .docx**：生成 Word 文档下载
- **文件直传模式**：输出与输入同格式的文件（.txt 或 .docx）

### 🔍 变更高亮

转换结果中，被修改的标点以高亮色标注，一眼看到哪些地方发生了变化。

### ⚠️ 未闭合引号检测

如果文本中的引号数量为奇数（未配对闭合），工具会在结果区域上方给出警告提示，并将最后一个未闭合的引号自动处理为后引号。

### 🚧 已知局限
 
暂不支持 `.wps` 格式文件的直接转换。`.wps` 是 WPS Office 的私有格式，网页端无法在保留排版的情况下直接处理。上传 `.wps` 文件时会弹窗提示，并提供两种替代方案：在 WPS Office 中另存为 `.docx` 后重新上传（推荐），或手动复制文本内容粘贴到输入框中进行转换。

## 工具使用方法

### 直接打开 HTML 文件

下载 `中文标点智能转换工具v1_1.html`，用浏览器打开即可使用，无需网络连接。


## 技术实现

- **纯前端单文件**：HTML + CSS + JavaScript，无需构建工具或后端服务
- **引号配对算法**：基于状态机的逐字符扫描，维护开闭状态进行配对，结合上下文语境判断
- **docx 处理**：使用 JSZip 解压 .docx 文件（本质是 ZIP 包），直接操作内部 XML 中的 `<w:t>` 文本节点进行标点替换，完整保留原文档的样式和排版
- **docx 导出**：使用 docx.js 生成新的 Word 文档
- **依赖库**（均通过 CDN 引入）：
  - [docx 8.5.0](https://github.com/dolanmiri/docx) — 生成 .docx 文件
  - [JSZip 3.10.1](https://stuk.github.io/jszip/) — 解析和生成 .docx/.zip 文件

## 浏览器兼容性

支持所有现代浏览器：Chrome、Firefox、Safari、Edge。移动端浏览器同样可用。

## 隐私说明

所有文本处理均在浏览器本地完成，不会将任何数据上传到服务器。
